## WHAT — 어떤 문제를 푸는가

### Problem Statement

> 반려동물 보호자는 진료 기록을 여러 곳에 흩어서 관리하기 때문에, 다음 접종일을 놓치거나 수의사 상담 시 제대로 이력을 전달하지 못한다.  
> 근본 원인: 반려동물 의료 기록에 특화된, 로그인 없이 즉시 쓸 수 있는 도구가 없다.

### 기존 대안이 실패하는 이유

| 대안 | 실패 이유 |
|------|-----------|
| 종이 수첩·영수증 | 분실·검색 불가, 상담 시 제시 어려움 |
| 일반 메모 앱 | 구조가 없어 날짜·항목 조회 불가 |
| 동물병원 앱 | 특정 병원 전용, 기록 이식 불가 |
| 종합 반려동물 앱 | 회원가입 필수, 과도한 기능으로 진입 장벽 높음 |

## HOW — 어떻게 해결하는가

### 핵심 기능 (MoSCoW)

| 분류 | 기능 |
|------|------|
| Must | 반려동물 프로필 등록 (이름·종·생년월일) |
| Must | 진료 기록 입력 (날짜·항목 필수 / 병원·비용·메모 선택) |
| Must | 접종 일정 자동 계산 (규칙 기반, 강아지·고양이) |
| Must | AI 요약 리포트 생성 (Claude API, 버튼 클릭 시에만) |
| Should | 리포트 PDF 다운로드 |
| Should | 재방문일 직접 입력·표시 |
| Won't | 로그인·계정 |
| Won't | 서버 DB 저장 |
| Won't | 푸시·이메일 알림 |
| Won't | 강아지·고양이 외 동물 |
| Won't | OCR 사진 입력 |

### MVP 정의

Must 항목 5개만으로 구성된 버전.  
데이터는 기기 localStorage에만 저장. 로그인 없음.

### 아키텍처 원칙

- 서버는 Claude API 키 보호 목적으로만 존재
- LLM 호출은 사용자의 명시적 버튼 클릭에서만 발생
- 동일 기록의 리포트는 localStorage에 캐싱 (불필요한 API 호출 방지)

## 제외 범위

MVP 이후로 명시적으로 미룬 것들:

- 로그인 / 다기기 동기화

- 다중 보호자 공유
