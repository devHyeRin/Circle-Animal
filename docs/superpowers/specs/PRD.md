# 반려동물 건강 스케줄러 — PRD (Product Requirements Document)

**날짜:** 2026-04-14  
**상태:** MVP 설계 확정  
**스택:** React (Vite) + Tailwind CSS / Node.js + Express  

---

## 1. 서비스 개요

반려동물 보호자가 진료 기록을 텍스트로 입력하면, AI가 다음 접종일·재방문 일정을 추천하고, 누적된 기록을 수의사 상담용 요약 리포트로 생성해주는 웹 서비스.

**핵심 가치:**
- 흩어진 진료 기록을 한 곳에서 관리
- 다음 일정을 놓치지 않도록 자동 계산
- 수의사 상담 시 바로 제출할 수 있는 리포트 생성

---

## 2. 타겟 사용자

- 강아지·고양이를 키우는 반려동물 보호자
- 진료 기록을 직접 관리하고 싶지만 별도 앱 설치는 번거로운 사람
- 수의사 방문 전 기록을 정리해야 하는 사람

---

## 3. MVP 범위

### 포함
- 반려동물 프로필 등록 (이름, 종류, 품종, 생년월일, 성별)
- 진료 기록 입력 (구조화 필드 + 자유 텍스트 메모)
- 접종·재방문 일정 자동 계산 (규칙 기반)
- 수의사 상담용 요약 리포트 생성 (LLM)
- 리포트 화면 표시 + PDF 다운로드
- 데이터 저장: localStorage (로그인 없음)
- 반응형 UI (모바일 브라우저 지원)

### 제외 (MVP 이후)
- 로그인 / 계정 관리
- 서버 사이드 데이터 저장 (DB)
- OCR (사진 촬영으로 기록 입력)
- 푸시 알림 / 이메일 알림
- 강아지·고양이 외 동물 지원
- 다중 보호자 / 공유 기능

## 4. 데이터 모델

모든 데이터는 클라이언트 localStorage에 저장합니다.

### Pet (반려동물 프로필)
```json
{
  "id": "uuid",
  "name": "초코",
  "species": "dog",           // "dog" | "cat"
  "breed": "말티즈",          // 선택 입력
  "birthDate": "2021-03-15",
  "gender": "female",
  "neutered": true,
  "createdAt": "2026-04-14T10:00:00Z"
}
```

### MedicalRecord (진료 기록)
```json
{
  "id": "uuid",
  "petId": "uuid",
  "date": "2026-04-14",                   // 필수
  "items": ["종합백신 5차", "켄넬코프"],  // 필수 — 진료/접종 항목
  "hospital": "행복동물병원",             // 선택
  "cost": 45000,                          // 선택 — 진료 비용 (원)
  "nextVisitDate": "2026-07-14",          // 선택 — 사용자가 직접 입력한 재방문일
  "memo": "식욕 감소 증상 언급. 2주 후 재확인 권고.",  // 선택 — 자유 텍스트
  "createdAt": "2026-04-14T10:00:00Z"
}
```

## 5. 핵심 기능

### 5-1. 접종 스케줄 계산 (규칙 기반)

`client/src/utils/scheduleRules.js`에서 처리.  
동물 종류별 표준 접종 스케줄을 정의하고, 마지막 접종일을 기준으로 다음 권장일을 계산.

**MVP 범위 명확화:**
- **접종 일정 자동 계산**: `species` + `항목명`으로 규칙을 매핑해 다음 접종 권장일을 자동 산출. 아래 표의 항목이 대상.
- **재방문 일정**: 자동 계산하지 않음. 사용자가 진료 기록 입력 시 직접 입력한 `nextVisitDate`를 저장하고 화면에 표시하는 수준.

**강아지 기본 접종 스케줄:**
| 항목 | 주기 |
|------|------|
| 종합백신 (DHPPL) | 1년 |
| 코로나 | 1년 |
| 켄넬코프 | 6개월~1년 |
| 광견병 | 1년 |
| 심장사상충 예방약 | 1개월 |
| 외부기생충 예방약 | 1개월 |

**고양이 기본 접종 스케줄:**
| 항목 | 주기 |
|------|------|
| 종합백신 (FVRCP) | 1년 |
| 광견병 | 1년 |
| 심장사상충 예방약 | 1개월 |

규칙 테이블에 없는 항목은 자동 계산 대상에서 제외되며, 사용자가 입력한 `nextVisitDate`가 있는 경우 해당 날짜를 그대로 표시.

**구조화 필드:**

| 필드 | 입력 방식 | 필수 여부 |
|------|-----------|-----------|
| 날짜 | date picker | 필수 |

| 비용 | 숫자 입력 (원) | 선택 |

**자유 텍스트 필드:**

| 필드 | 입력 방식 | 필수 여부 |
|------|-----------|-----------|
| 메모 | textarea | 선택 — 특이사항, 수의사 코멘트 등 |

### 5-3. LLM 요약 리포트 생성

**흐름:**
1. 사용자가 "리포트 생성" 버튼 클릭
2. 클라이언트가 펫 프로필 + 전체 진료 기록을 서버로 전송
3. 서버(`POST /api/summary`)가 LLM에 프롬프트 전송
4. LLM 응답을 클라이언트로 반환, 화면에 표시

**프롬프트 구성 (서버 사이드):**
```
당신은 수의사 상담 보조 AI입니다.
아래 반려동물의 진료 기록을 바탕으로,
수의사가 빠르게 파악할 수 있는 요약 리포트를 작성하세요.

[반려동물 정보]
이름: {name}, 종: {species}, 품종: {breed}, 나이: {age}

[진료 기록]
{records}

[출력 형식]
1. 전체 건강 이력 요약 (3~5문장)
2. 주요 접종 현황
3. 특이사항 및 수의사 권고 사항
4. 다음 권장 방문 일정
```

**LLM 선택:** Claude API (Anthropic) — `ANTHROPIC_API_KEY`는 서버 환경변수로 관리.

**LLM 사용 원칙:**
- 호출은 사용자가 "리포트 생성" 버튼을 명시적으로 클릭할 때만 발생. 자동 호출·백그라운드 호출은 없음.
- 동일한 데이터로 반복 생성을 방지하기 위해, 마지막 생성 시각과 결과를 localStorage에 캐싱. 기록 변경이 없으면 캐시된 결과를 우선 표시하고 재호출 여부를 사용자가 선택.
- MVP 단계에서는 별도의 Rate Limit 미들웨어 없이 위 캐싱 전략으로 사용량을 최소화. 필요 시 서버에 호출 횟수 로깅을 추가해 모니터링 기반 마련.

---

### 5-4. 리포트 출력 & PDF 다운로드

- 화면: `ReportViewer` 컴포넌트로 섹션별 정리된 뷰 표시
- PDF: `jsPDF` + `html2canvas`로 뷰를 그대로 PDF 변환
- 파일명: `{petName}_health_report_{YYYY-MM-DD}.pdf`

## 6. API 명세

### POST /api/summary

**Request:**
```json
{
  "pet": { ...Pet },
  "records": [ ...MedicalRecord[] ]
}
```

**Response (200):**
```json
{
  "summary": "마크다운 형식의 요약 텍스트"
}
```

**Response (500):**
```json
{
  "error": "LLM 호출 실패 메시지"
}
```

### GET /api/health
서버 상태 확인용. `{ "status": "ok" }` 반환.

## 8. 확장성 고려사항

다음 기능은 MVP에서 제외하되, 아래 방식으로 확장 가능하도록 설계:

| 향후 기능 | 확장 방법 |
|-----------|-----------|
| 로그인 / 계정 | `useStorage` 훅을 API 기반으로 교체, 서버에 DB 추가 |
| 새 동물 추가 | `scheduleRules.js`에 `species` 항목 추가 |
| 알림 기능 | 서버에 cron job + 이메일/푸시 발송 추가 |
| OCR 입력 | 별도 엔드포인트로 이미지 → 텍스트 변환 후 기존 폼 연결 |
